# Решения и сложности

## Принятые решения

1. Выбран веб-стек (`Vite + Vanilla JS`) для максимально быстрого MVP и простого деплоя на статический хостинг.
2. Сделан mobile-first вертикальный UI, чтобы запись экрана выглядела как готовый TikTok/Reels-креатив без доп. адаптации.
3. Добавлен встроенный `Face Cam` overlay внутри приложения, чтобы получить формат "экран + фронталка" прямо в записи.
4. Реализована fuzzy-проверка ответа (Levenshtein + нормализация текста), чтобы смягчить ошибки распознавания речи.
5. Звуки сделаны через Web Audio API (без внешних ассетов), чтобы MVP работал из коробки.

## Сложности и как решены

1. **Разная поддержка Speech API в браузерах.**
   - Добавлен fallback через ручной ввод.
   - В документации отмечены ограничения и рекомендованные условия запуска.
2. **Нестабильность voice-транскрипта (ошибки распознавания).**
   - Введен допуск к небольшим опечаткам.
   - Разрешены синонимы через массив `answers`.
3. **Требование "вирусного" визуала для записи экрана.**
   - Добавлены акцентные анимации, контрастные цвета, счетчик streak, быстрые фидбек-реакции, конфетти и SFX.
